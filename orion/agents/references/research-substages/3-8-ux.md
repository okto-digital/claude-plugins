## Purpose

Two distinct lenses working together. The **behavioural UX** lens analyses how sites work — navigation, information architecture, conversion flows, mobile patterns — grounded in persona journey maps from R5 and trust requirements from R6. The **visual UI** lens analyses how sites look — colours, typography, imagery, spacing, brand consistency — across client, all roster competitors, and reference sites.

All roster competitors are included (vs direct threats only in other substages) because visual benchmarking needs the broadest sample to identify both industry patterns and differentiation opportunities.

**For new builds (site_type = "new"):** No client site to analyse. The stage becomes purely a competitor and reference benchmark — establishing the UX and visual standards the new build should meet or exceed.

## Upstream Carry-Forward

R5 provides the primary design target, persona device preferences, journey stage expectations, and trust thresholds — these form the evaluation criteria for all UX analysis. R6 provides which trust signals need prominent placement and what messaging resonates. R7 provides performance scores and their experience implications (a 4s LCP means users stare at a blank page — that's a UX finding, not just a metric) plus accessibility scores. R4 provides table-stakes vs differentiating functionality expectations. Read all before starting.

## Minimum Scope

Cover at least these areas. You may go beyond them if evidence warrants it.

- Evaluation framework — persona-derived criteria established before any analysis: primary design target, device weighting, journey stage requirements, trust signal needs. Prevents generic UX opinions
- Navigation and information architecture — nav type, depth, labelling (customer language vs jargon), mobile nav, content hierarchy, path tracing for primary persona from landing to goal
- Conversion flows and CTAs — primary CTA placement and consistency, form design (fields, steps, error handling), friction points, secondary conversion paths, trust signal placement near decision points. Cross-referenced with R5 persona journey maps
- Mobile UX — responsive behaviour, touch targets, content priority on mobile, mobile-specific patterns (click-to-call, maps, simplified forms). Weighted by R5 persona device preferences
- Usability heuristics — Nielsen's 10 as reference lens (not rigid checklist), plus practical accessibility beyond R7's technical WCAG scan (readable font sizes, obvious clickable elements, colour-blind safe, logical reflow)
- Visual inventory per site — colour (hex values where detectable), typography (families, weights, sizes), imagery style, layout and spacing (grid, density, white space), motion and interaction, brand consistency across pages
- Visual pattern analysis — industry visual language (dominant palette, layout approach, imagery style), conformity vs differentiation map (where competitors look the same = convention, where they diverge = opportunity, where no one invests = standout potential)
- Gap classification — Convention (all share it, client deviates — adopt), Best Practice (top performers do it — recommend), Whitespace (no one does it well — differentiate), Client Strength (client excels — preserve)

## Methodology — Processing Sequence

Eight steps. Step 1 builds the evaluation framework. Steps 2-5 analyse behavioural UX. Steps 6-7 analyse visual UI. Step 8 synthesises across both lenses.

Dispatch web-crawler for each site before analysis.

### Behavioural UX — how sites work

**Step 1 — Evaluation framework setup:** Before analysing any site, establish evaluation criteria from prior research. From R5: primary design target (this persona's needs take priority), device preferences (determines mobile UX weighting), journey stage expectations (what the site should do at each stage), trust thresholds (what must be visible and where). From R6: which trust signals need prominent placement, what messaging resonates. From R7: performance scores and their experience implications, mobile readiness, accessibility issues creating real usability barriers. From R4: table-stakes functionality, differentiating functionality, local-vs-global gap in UX patterns.

**Step 2 — Navigation and information architecture:** Per site: navigation type (horizontal/hamburger/mega menu/sidebar), depth (clicks from homepage to deepest service page), labelling (customer language vs internal jargon), utility navigation (search, language switcher, account, cart), footer navigation, mobile navigation (mirrors desktop or adapts — sticky, bottom nav, standard hamburger). Information architecture: how services/products are organised (flat vs hierarchical, by category vs audience vs use case), homepage content hierarchy (what's above the fold, scroll-order story), path tracing for primary persona (how quickly can they find what they need). Cross-site patterns: universal structures (table-stakes) vs divergent structures (design choices).

**Step 3 — Conversion flows and CTAs:** Per site, trace the primary conversion path from homepage to main conversion action. Primary CTA: what, where (header/hero/inline/footer/sticky), prominence. CTA consistency across pages. Form design: field count, required vs optional, multi-step or single, error handling. Friction points: where might the primary persona hesitate (too many fields, unclear next step, missing trust signals at decision point, no pricing when expected). Secondary conversion paths (newsletter, download guide, free consultation). Trust signal placement: are R6's credibility markers positioned near CTAs, on pricing pages, adjacent to forms? Cross-reference with R5 journey maps: does each stage provide what the persona needs?

**Step 4 — Mobile UX:** Weighted by R5 persona device preferences — if primary persona is mobile-dominant, this gets proportionally more attention. Responsive behaviour (genuinely adapts or just shrinks), touch targets (sized for thumb interaction), content priority on mobile (same hierarchy or reorganised), mobile-specific patterns (click-to-call, maps, simplified forms, thumb-zone navigation), performance perception (using R7's mobile data — does hero load before CTA, does navigation work immediately).

**Step 5 — Usability heuristics and practical accessibility:** Using Nielsen's 10 as reference lens, flag notable patterns: visibility of system status (loading states, confirmations, progress), match between system and real world (language matches how audience speaks per R6's customer voice), user control and freedom (easy back-navigation, undo), consistency within site and across industry, error prevention in forms, recognition over recall. Practical accessibility beyond R7's technical scan: readable font sizes at actual usage, obviously clickable/tappable elements, colour-coded elements working without colour, logical reading order on mobile reflow. This is qualitative UX assessment — R7 found violations, this step evaluates whether the site feels accessible to use.

### Visual UI — how sites look

**Step 6 — Visual inventory:** Per site, capture concrete specifications where detectable. Colour: primary/secondary/accent (hex values), backgrounds, contrast approach, usage pattern (branding/wayfinding/emphasis/sparse). Typography: heading and body font families and weights, size range, type scale, line height and spacing. Imagery: photography style (studio/lifestyle/abstract/stock/custom/AI), quality and consistency, hero approach (photo/illustration/gradient/video/animation), icon style. Layout and spacing: grid system (columns, max width, margins), density (compact vs spacious), section rhythm (alternating backgrounds, cards, full-width bands), white space. Motion: page transitions, scroll animations, micro-interactions, loading animations. Brand consistency: do colour/typography/imagery stay consistent across page types or feel like different eras of design.

**Step 7 — Visual pattern analysis:** Synthesise across all sites. Industry visual language: dominant colour palette, layout approach, imagery style for this market. Conformity vs differentiation map: where most competitors look the same (visual conventions — breaking them is a risk), where they diverge (differentiation opportunities), where no one invests (potential standout elements). Client position: does the client's visual identity align with or diverge from the pattern, visually dated or ahead of competitors, visual strengths worth preserving in redesign.

### Synthesis

**Step 8 — Cross-lens gap analysis:** Synthesise across both lenses — don't restate per-site findings.

Behavioural UX gaps: navigation/IA patterns client lacks that competitors share, conversion flow weaknesses relative to best practices, mobile experience gaps (weighted by persona device preference), journey stage gaps where the client fails to serve the persona.

Visual UI gaps: where client falls below industry standard, where visual landscape is homogeneous (opportunity to stand out), where reference sites set a higher bar than any competitor.

Combined findings: patterns that combine UX and UI insights (e.g. "competitors with cleaner visual design also have simpler conversion flows"), conflicts between UX best practice and visual trends (e.g. "industry trend is dense layouts but persona research suggests primary audience prefers simplicity").

Classify each gap: Convention (all or most competitors share this pattern, client deviates — adopt to avoid confusing users), Best Practice (top performers do this, others don't — recommend, likely contributes to their performance), Whitespace (no competitor does this well — differentiate, highlight in proposal as competitive advantage), Client Strength (client does this better — preserve and carry forward into new design).
